# פרומפטים מוכנים להפעלה

להדבקה ישירה לספריית הפרומפטים. הטקסטים כתובים כך שהסוכן יפעל רק דרך הקבצים והסקריפטים הקיימים, ללא פקודות גולמיות.

---

## בדיקות מקדימות לפני פעולה לוגית

* לקרוא את AGENT_CONFIG_TEMPLATE.yaml
* לאמת שכל הערכים הנחוצים קיימים ואם חסר ערך לעצור ולבקש הבהרה
* לפעול לפי הפרומפט התואם למשימה בלבד
* לא להפעיל פקודות Git גולמיות ולהשתמש רק בסקריפטים שבתיקיית automation git
* לעדכן רק קבצים בתיקיות שמותרות לפי allowedPaths

---

## פרומפט הפעלה: פתיחת מחזור חדש ויצירת לוג לפי מחזור

> יש להריץ קודם את: בדיקות מקדימות לפני פעולה לוגית 

### מטרה  
פתיחת מחזור עבודה חדש והכנת לוגים ותקציר לפי ההגדרות בקובץ AGENT_CONFIG_TEMPLATE.yaml.

### הוראות
1. קרא את הקובץ AGENT_CONFIG_TEMPLATE.yaml ומצא את logging.basePath ואת logging.rotationPolicy ואת logging.summaryTemplatePath.
2. אם rotationPolicy שונה מ by_cycle עצור ובקש אישור להמשך. אם הערך הוא by_cycle המשך.
3. גזור מזהה מחזור חדש. אם מוגדר מזהה ידוע במסמכי planning או ב branch הנוכחי, השתמש בו. אם לא, הצע מזהה בסגנון CYCLE_YYYYMMDD_HHMM.
4. צור קובץ לוג חדש בשם הבא בתוך logging.basePath
   IMPLEMENTATION_LOG_<cycle_id>.md
   אם יש לוג פתוח ממחזור קודם, השאר אותו ללא שינוי.
5. הוסף כותרת ללוג החדש עם מזהה המחזור ותאריך ושדות ריקים ל פעולות תוצאות קישורים.
6. הפק תקציר ראשוני לפי התבנית שב summaryTemplatePath ושמור בתיקיית logs בשם
   IMPLEMENTATION_LOG_SUMMARY_<cycle_id>.he.md
   מלא רק את שדות הכותרת והמטה. אל תמציא תוכן.
7. דווח בסיום מה נוצר, עם נתיבים מדויקים לקבצים.

### כללים
אל תריץ פקודות Git גולמיות. אם צריך commit השתמש בסקריפט automation/git/STAGE_AND_COMMIT.sh בלבד.

---

## פרומפט הפעלה: רוטציה ידנית של לוגים לפי מחזור קיים

> יש להריץ קודם את: בדיקות מקדימות לפני פעולה לוגית 

### מטרה  
סגירת לוג המחזור הנוכחי ופתיחת לוג למחזור הבא, כאשר rotationPolicy הוא by_cycle או manual.

### הוראות
1. קרא את AGENT_CONFIG_TEMPLATE.yaml. אם rotationPolicy הוא monthly עצור ובקש אישור.
2. קבע מי הוא ה cycle הנוכחי מתוך שם קובץ הלוג הפתוח או מתוך תוכן הקובץ האחרון בתיקיית logs.
3. סגור את הלוג הנוכחי על ידי הוספת חתימת זמן ושדה Status סגור.
4. הפק תקציר מעודכן לקובץ
   IMPLEMENTATION_LOG_SUMMARY_<cycle_id>.he.md
   מלא רק סטטוס ותאריכים. אל תסכם תוכן טכני ללא מקורות.
5. פתח לוג חדש למחזור הבא בשם
   IMPLEMENTATION_LOG_<new_cycle_id>.md
   עם כותרת ותבנית שדות ריקה.
6. דווח מה נסגר ומה נפתח והחזר נתיבים לקבצים.

## כללים
אין פקודות Git ישירות. אם יש צורך בקומיט, קרא ל automation/git/STAGE_AND_COMMIT.sh.

---

## פרומפט הפעלה: רישום פעולה ללוג המחזור

> יש להריץ קודם את: בדיקות מקדימות לפני פעולה לוגית 

### מטרה  
רישום פעולה חדשה בקובץ הלוג של המחזור הפעיל.

### הוראות
1. אתר את קובץ IMPLEMENTATION_LOG_<cycle_id>.md האחרון בתיקיית logs לפי סדר זמן.
2. הוסף רשומה חדשה בתחתית במבנה הבא
   תאריך ושעה
   מבצע סוכן או אדם
   קבצים שהושפעו רשימת נתיבים ביחס לשורש הריפו
   תיאור קצר של הפעולה
   תוצאה הצלחה או כישלון
   מזהי קומיט אם קיימים
3. אל תיצור שדות שאינם קיימים בתבנית הלוג.
4. שמור את הקובץ והחזר את הנתיב ששונה.

## כללים
אם נדרש קישור למחזור בתקציר, אל תעדכן ידנית את התקציר. עדכון תקציר מתבצע בפרומפט הייעודי.

---

## פרומפט הפעלה: עדכון תקציר מחזור מתוך לוג

> יש להריץ קודם את: בדיקות מקדימות לפני פעולה לוגית 

### מטרה  
עדכון קובץ התקציר של המחזור על בסיס נתונים מתוך IMPLEMENTATION_LOG_<cycle_id>.md.

### הוראות
1. אתר את קובץ התקציר של המחזור
   IMPLEMENTATION_LOG_SUMMARY_<cycle_id>.he.md
   אם אינו קיים הפק אותו מהתבנית ב summaryTemplatePath.
2. סכם את המידע לפי השדות שמופיעים בתבנית בלבד. אין להוסיף שדות חדשים.
3. ציין מספר פעולות בוצעו הצלחות כשלים פריטים פתוחים.
4. שמור את הקובץ והחזר את הנתיב.

## כללים
אל תנסח תוכן שלא קיים בלוג. כל משפט חייב להישען על רשומות קיימות.

---

## פרומפט הפעלה: קומיט עם קישור ללוג המחזור

> יש להריץ קודם את: בדיקות מקדימות לפני פעולה לוגית 

### מטרה  
יצירת קומיט עומד מדיניות עם קישור ללוג המחזור.

### הוראות
1. קרא את AGENT_CONFIG_TEMPLATE.yaml ומצא את git.allowedPaths ואת commitMessage.includeCycleLogLink.
2. אמת שכל הקבצים ששונו נמצאים בתוך הנתיבים המותרים. אם לא עצור ובקש אישור.
3. בנה הודעת קומיט קצרה עם תגים מקוצרים אם includeTags הוא true.
4. אם includeCycleLogLink הוא true הוסף בסוף ההודעה שורת טריילר
   Relates-to-log docs/logs/IMPLEMENTATION_LOG_<cycle_id>.md
5. קרא לסקריפט automation/git/STAGE_AND_COMMIT.sh בלבד עם ההודעה שנבנתה.
6. דווח על הודעת הקומיט שהשתמשת בה ועל הקבצים שנכללו.

## כללים
אין שימוש בפקודות git ישירות. אין דחיפה. אין פתיחת PR במסגרת הפרומפט הזה.
